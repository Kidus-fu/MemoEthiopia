Overview

Memo Ethiopia is a real-time note-taking application designed to help users create, manage, and share notes efficiently. Built with Django and Django REST Framework (DRF), it offers a robust backend architecture for handling user authentication, note management, and real-time data interaction.

This documentation provides a comprehensive overview of the backend implementation, including:

  • RESTful APIs to perform CRUD operations on notes and user data
  • Authentication & Authorization using token-based access control
  • Error Handling & Validation to ensure data integrity
  • Deployment Instructions for running the backend on production servers in render.com

Memo Ethiopia is built with scalability and extensibility in mind, making it suitable for both individual users and teams.

Key Backend Features

  • Authentication – Secure login, registration, and token-based access control
  • Note Management – Create, edit, delete, and organize notes with features like:
    • Folders for categorizing notes
    • Trash for soft-deletion and recovery
    • Pinning for marking important notes
    • Archiving for decluttering the workspace
  • Custom Colors – Allow users to color-code their notes
  • AI Agent Integration – Enable smart note suggestions or actions via an AI assistant
  • OTCB Chat Functionality – One-Time Chat Bot for quick chats
  • Translation Services – Translate words between languages (e.g., am and en) for multilingual support

Technologies Used

  • Django (Web framework)
  • Django REST Framework (DRF) (API development)
  • PostgreSQL (Database)
  • DRF-YASG (Swagger API documentation)
  • JWT Authentication (User authentication)
  • Django CORS Headers (Cross-Origin Resource Sharing)
  • Langchain (AI chat and Agent)
  • Groq (LLM integration)
  • Gunicorn (WSGI HTTP server for UNIX)

Installation

Follow these steps to set up the backend:

  1. Clone the repository: 
    git clone https://github.com/Kidus-fu/MemoEthiopia.git
  2. Create a virtual environment and activate it: 
    python -m venv venv
source venv/bin/activate  # On Windows use venv\Scripts\activate
  3. Install dependencies: 
    pip install -r requirements.txt
  4. Set up the database: 
    python manage.py migrate
  5. Create a superuser: 
    python manage.py createsuperuser
  6. Run the development server: 
    python manage.py runserver

Endpoints

Authentication & Authorization

  • POST /api-v1/username/token/ → Obtain JWT token using username
  • POST /api-v1/email/token/ → Obtain JWT token using email
  • POST /api-v1/token/refresh/ → Refresh JWT token
  • POST /api-v1/register/ → Sign up a new user

User Endpoints

  • GET /api-v1/users/ → List all users
  • POST /api-v1/register/ → Register a new user
  • GET /api-v1/users/{uuid}/ → Retrieve user details

Notes Endpoints

  • GET /api-v1/notes/ → List all notes
  • POST /api-v1/notes/ → Create a new note
  • GET /api-v1/notes/{uuid}/ → Retrieve a specific note
  • PUT /api-v1/notes/{uuid}/ → Update an existing note
  • PUT /api-v1/notes/outtrash/{uuid}/ → Restore a note from trash
  • DELETE /api-v1/notes/{uuid}/ → Delete a note

Deployment

Environment variables should be set in the .env file:


            ALLOWED_HOSTS=*
            DATABASE_URL=postgres://user:password@localhost/dbname
            SECRET_KEY=your_secret_key
            JWT_SECRET_KEY=your_jwt_secret_key

To deploy the backend on Render, follow the instructions on their website and connect your GitHub repository.
